The shortlist (everything below in one breath)

Vercel + Next.js + Sanity + HubSpot Service Hub (with custom call routing) + CallRail + Vercel Edge A/B + GA4 + Heap + Ahrefs + Screaming Frog. Total tooling cost ~$1,200/mo at MVP, scaling toward ~$3,500/mo by Phase 3 as call volume grows. Engage Klein Moynihan Turco or Eckert Seamans for TCPA review at $20–40K for a launch-ready review.

1. Hosting & redirect layer

Pick
Vercel (Pro plan)
Pairs with Next.js · edge redirects native · global CDN
Vercel is the right call here for three concrete reasons: (1) backlink migration redirects can live in vercel.json as 301s with no extra infrastructure, which is the single biggest equity preservation lever per Section 4 of the spec; (2) edge-rendered Next.js gives sub-2s LCP on 3G mobile (Section 18 performance target); (3) Vercel Edge Config supports A/B testing without a third-party dependency, removing one vendor entirely.
~$20–80/mo per seat at MVP, scaling with bandwidth
Considered: Cloudflare Pages (cheaper, but Workers complicate CMS data-fetch patterns); Netlify (similar but smaller Next.js community); AWS Amplify (overkill for a content site).

2. Frontend framework

Pick
Next.js 15 with App Router
SSR-first React · ISR for state/city pages · server actions for the calculator
Next.js is the de-facto choice for a content-led SEO site that needs both static rendering (state pages, articles) and a dynamic surface (calculator, funnel). ISR (incremental static regeneration) lets you regenerate the FPL/SMI-driven state pages annually when HHS publishes new poverty thresholds without a full rebuild. Server actions handle the calculator POST /api/eligibility/calculate endpoint without separate API infrastructure.
Open source · runs on Vercel Pro pricing above

3. CMS

Pick
Sanity
Headless · structured content · GROQ for query flexibility
Sanity wins on three things specific to this build: (1) structured fields handle the per-page-type schemas (Article + FAQPage + GovernmentService) cleanly, including the per-data-point "last verified" CMS field the spec calls for; (2) the data model in Section 15 (states_data, FPL/SMI tables, eligibility rules-as-data) maps directly to Sanity document types — eligibility logic stays as config, not code, exactly as the spec requires; (3) Portable Text gives writers a real-time-collaborative editor that doesn't break the schema on a missed paste.
Free for <3 users · $99/mo for Growth · $949/mo at scale
Considered: Contentful (more expensive, less flexible queries); Payload (great but self-hosted means more ops); WordPress (no, this is a YMYL site and structured content is non-negotiable).

4. CRM & call center routing

Pick
HubSpot Service Hub Pro + custom queue logic via webhooks
Native lead capture · disposition-aware reporting · Twilio Flex for the actual phone routing
The spec explicitly flags CRM as the biggest open dependency. HubSpot is the right starting point because: (1) the lead schema (UTM + click ID + landing variant + quiz path + consent record) maps to HubSpot custom properties without engineering work; (2) attribution and source rollup come built-in, hitting the Section 19 dashboard requirements without a Looker build; (3) the ecosystem of pre-built TCPA-compliance integrations (e.g., Convoso, PhoneBurner) makes it easy to swap in a dedicated dialer when call volume justifies it.
For the actual phone routing — priority/standard/specialist/crisis queues — pair HubSpot with Twilio Flex for the call control plane (queue logic, IVR, agent UI) and webhook lead-payload to it. This separates "lead system of record" from "call routing engine," which lets you change either independently.
HubSpot Service Hub Pro: $450/mo · Twilio Flex: $1/active hour per agent
Considered: Five9 (mature but heavy and slow to integrate); CallTools (good for basic dialing, weak on attribution); custom build (defer until volume justifies it — at least 12 months out).

5. Dynamic number insertion (DNI)

Pick
CallRail
Per-source / per-campaign / per-CTA-placement number insertion
CallRail is the call-tracking incumbent for content-led lead-gen and integrates natively with HubSpot, GA4, Google Ads, and Meta CAPI. The spec calls for DNI at four levels of granularity (source, campaign/keyword, landing variant, CTA placement) — CallRail supports all four out of the box with no engineering. Its Conversation Intelligence (transcription + scoring) becomes useful in Phase 2 for testing call-to-conversion correlations across content types.
Starting $50/mo · scales to $300+/mo at high call volume (most of cost is per-minute)
Considered: Invoca (more expensive, better at enterprise integrations — overkill at MVP); Twilio-built (more flexible but 10× the engineering investment to ship the same attribution).

6. A/B testing

Pick
Vercel Edge Config + Statsig (free tier)
Edge-rendered variants · zero client flicker · sequential testing
A/B testing on the funnel page must be edge-rendered, not client-side, because the conversion event (click-to-call) often fires before client JS hydrates. Vercel Edge Config + Statsig solves this without paying Optimizely or VWO prices. Statsig also gives you sequential testing (you can stop a test as soon as significance is reached, not at a pre-set date) which matters for the spec's 2-week-minimum testing discipline.
Free at 1M events/mo · ~$200/mo at typical paid-funnel scale
Considered: Optimizely ($1,000+/mo, enterprise); VWO ($300+/mo, weaker on edge testing); GrowthBook (good but more setup).

7. Analytics stack

Pick
GA4 (free) + Heap (Pro)
GA4 for source attribution · Heap for behavioral analysis
GA4 alone is insufficient for the event taxonomy in Section 19 (per-step abandonment, time-on-step, per-CTA-placement segmentation) — the cardinality of custom dimensions hits limits fast. Heap handles the autocaptured behavioral side without instrumentation tax: every quiz step, every click, every form-field abandonment is tracked by default. You then send a small set of canonical conversion events (qualified, lead_submitted, call_connected) to GA4 for paid-channel attribution. This split keeps GA4 clean for Google Ads optimization while giving the team real product analytics.
GA4: free · Heap: $0 free tier, ~$3,600/yr at meaningful volume
Considered: Mixpanel (similar to Heap, slightly weaker autocapture); Amplitude (great but enterprise-priced); just GA4 (will hit limits within Phase 2).

9. TCPA-experienced legal counsel

Recommended firms (any of three)
Klein Moynihan Turco · Eckert Seamans · ZwillGen
All three have done TCPA work for performance marketers and lead-gen sites at scale
For a TCPA-heavy lead-gen site, you want a firm that has actually litigated TCPA class actions, not just a tech-friendly general-counsel-of-record. Klein Moynihan Turco is the deepest in this niche; Eckert Seamans handles a lot of insurance/financial lead-gen TCPA; ZwillGen is the better choice if you also want their privacy/data-protection bench (CCPA, CPRA, GPC). Florida Mini-TCPA review is a separate engagement — anyone of the three handles it but ask explicitly.
$20–40K for a launch-ready review (TCPA + Mini-TCPA + FTC disclaimers + consent record requirements). Quarterly retainer thereafter at $3–8K depending on activity.

10. Consent record storage & retention

Pick
PostgreSQL on Vercel Postgres + 7-year retention policy
Append-only consent log · queryable by phone, IP, timestamp · backed up to S3
Consent records (timestamp, IP, UA, exact consent language version, page URL) are TCPA litigation insurance — the FCC and plaintiffs' attorneys both expect immediate retrieval. Storing in Postgres alongside leads keeps everything in one place; a 7-year retention policy covers the standard TCPA statute-of-limitations (4 years) plus a safety margin. Version the consent language as a separate table and reference it by ID in each consent record so language changes are auditable.
~$25/mo at MVP volume; ~$200/mo by Phase 3

What's NOT decided

Three open items the spec calls out that I'm flagging rather than deciding because they're business calls, not technical:

  1. Spanish at MVP vs week 8. Spanish is essential for CA/TX/FL traffic but doubles compliance review cost. Recommendation: ship English MVP, target Spanish at week 6–8 once consent language and disclaimers have been pressure-tested in English first. This is a regret-minimization call the founders should make explicitly.
  2. Email capture: yes for nurture, or phone-only model? Recommendation: optional email field on the lead form, never required. Don't add an email-only nurture funnel until Phase 3 — call-first is the brand and email-nurture changes the funnel math.
  3. Call center hours. Recommendation: M-F 8a–8p ET + Sat 9a–5p ET to start; expand to 24/7 crisis line in Phase 2 once volume data justifies. Until then, after-hours users get scheduled-callback flow with SMS confirmation.

This document is a starting position, not a final architecture. Every recommendation should survive a 60-minute pressure test with the eventual hire who'll own it. The migration owner will likely have opinions on tooling; let them. The TCPA counsel will likely have opinions on consent storage; let them. The point of this doc is to make the build start tomorrow rather than 30 days from tomorrow.
